# Plots & Tables registry

Every artifact, keyed by **filename**: what it shows, its data source, how to
read it, and the takeaway. Regenerate with `python run_all.py` (figures →
`figures/*.pdf`, tables → `tables/*.csv`).

**Models.** `EDR` (analytical rigid-body dynamics + structured learned
corrections, **13.4 k** params), `Physics-Reg.` (black-box net + physics loss,
**549 k**), `FNN` (plain MLP, **70 k**). Full-data champion test RMSE
(trajectory-macro, 87 596-sample test split):
**Physics-Reg. 0.0896 ≤ EDR 0.0902 ≤ FNN 0.0972 N·m**. EDR's case is
**efficiency, not a headline win**: it lands within ~0.001 N·m of the best
baseline using ~40× fewer parameters than Physics-Reg. and ~5× fewer than FNN,
holds up best as training data shrinks (fig02), and keeps the smallest
generalization gap (fig03).

**Data sources.** The run record is the grid CSVs, regenerated by
`run_journal_grid_3model.py` reconstruct (disk-driven — every trained run):
`grid_results.csv` = the full-data architecture/HP sweep (**180 runs**: EDR 72 /
FNN 60 / PhysReg 48); `grid_results_dataeff.csv` = the data-efficiency sweep (30
runs over fractions 0.1–1.0). `shared/dataio.py` joins each grid row to its
on-disk model directory (`EDR/`, `FNN/`, `PhysicsRegularized/`) for weights +
per-joint metrics + training history. (EDR was swept over a wider HP range than
the baselines, so its grid distribution is the widest; its champion is still
~best-baseline.) `models_registry.yaml` is **not** used by the plots.

**Champion selection.** Each architecture's "champion" (used by figs 05–11, the
heatmaps, the trajectory plot and `headline_metrics.csv`) defaults to the best
**full-data** run by **test** RMSE. Switch the basis per run:
`--config-override champion_basis=val` (also `train`, `global`) and
`champion_full_data_only=false`. `tables/champion_selection.csv` reports the
winner under **all four** bases (global / train / val / test) side by side.

**Conventions.** SciencePlots theme (`pip install SciencePlots`, required). One
chart per file. No titles / `(a)(b)` letters (multi-panel identity is the axis
label). Non-bold Times New Roman throughout (STIX for math glyphs). One
horizontal frameless legend strip above the plot. EDR is red, heavier, drawn on
top (a focus choice, not a claim of dominance). **Every figure file opens with a
`TWEAKABLES` block** — edit those module-level variables to restyle that one
figure (sizes, colours, fonts, legend, smoothing, …).

**Savitzky–Golay.** On by default and *replacing* the raw line in every
series/curve/trajectory figure (fig02, fig03, fig05, fig06, fig07, fig10).
Disable: `python run_all.py --config-override savgol_enabled=false` (also
`savgol_window=`, `savgol_polyorder=`). Scatter / violin / heatmap figures are
never smoothed. **Tables always contain raw, unsmoothed data.**

## Figures

### `fig01_grid_rmse_distribution.pdf`
Violin + jittered points of test RMSE over each family's **best 50** full-data
grid runs (ranked by test RMSE); black bar = median, star = champion
(annotated). *Data:* `grid_df()` (`grid_results.csv`, `status==ok`). *Read:*
lower & tighter = better. *Takeaway:* trimming to the top 50 keeps the families
comparable; EDR's champion (star) reaches Physics-Reg.'s low end at ~40× fewer
params, FNN sits clearly higher. *Table:* `grid_summary.csv`, `grid_runs.csv`
(full, untrimmed).

### `fig02_data_efficiency_rmse.pdf`
Test RMSE vs % training data, one line/arch (savgol). *Data:* `sweep_df()`.
*Read:* low & flat = data efficient. *Takeaway:* EDR stays low at 10 % data
where FNN degrades. *Table:* `data_efficiency.csv`.

### `fig03_generalization_gap_vs_fraction.pdf`
(test − val) RMSE vs % training data (savgol). *Data:* `sweep_df()`. *Read:*
near zero = generalizes. *Takeaway:* EDR's gap is the smallest at every budget.
*Table:* `data_efficiency.csv`.

### `fig04_capability_radar.pdf`
Spider/radar over five per-architecture scalars **no other figure shows**:
pooled RMSE, R² overall, mean MAE, mean NRMSE, and parameter efficiency. Every
axis is oriented outward = better; the four accuracy/fit axes use ratio-to-best,
the parameter axis a log score (it spans ~40× vs <11 % for the others). Each
axis is annotated with its real best value. *Data:* `champion_results()` +
`param_count()`. *Read:* larger enclosed area = better all-round; the param axis
is the differentiator. *Takeaway:* Physics-Reg. edges the accuracy axes but
collapses toward the centre on size, while EDR nearly matches it everywhere at
~40× (vs Physics-Reg.) / ~5× (vs FNN) fewer parameters. *Table:*
`capability_profile.csv`.

### `fig05_train_val_curves.pdf`
Per-epoch train (dashed) vs val (solid) RMSE, 3 champions (savgol). *Data:*
champion `training_history.csv`. *Read:* curves together = healthy. *Takeaway:*
FNN over-fits; EDR's curves stay close. *Table:* `training_curves.csv`.

### `fig06_overfitting_gap_vs_epoch.pdf`
(val − train) RMSE vs epoch, 3 champions (savgol) — isolates the divergence
*trend* fig05 only shows implicitly. *Read:* growing = over-fitting. *Takeaway:*
EDR's gap stays small/flat. *Table:* `training_curves.csv`.

### `fig07_edr_correction_evolution.pdf`
EDR |δg|, ‖δM‖_F, |δC·q̇|, |δτ_f| vs epoch, log y (savgol). *Data:* EDR
champion history delta columns. *Read:* which analytical term EDR corrects.
*Takeaway:* interpretability the baselines lack. *Table:*
`edr_correction_evolution.csv`. **EDR-only.**

### `fig08_per_joint_rmse_heatmap.pdf`
Architecture × joint RMSE heatmap (compact, square cells; arch on the row labels
— no Y title); **green = best (low), red = worst**, normalized globally; cells
annotated; colorbar best↔worst. *Data:* champion predictions. *Table:*
`per_joint_metrics.csv`.

### `fig09_per_joint_r2_heatmap.pdf`
Architecture × joint R² heatmap (compact, square cells; no Y title); **green =
best (high), red = worst**, global normalization, annotated. *Data:* champion
predictions. *Table:* `per_joint_metrics.csv`.

### `fig10_trajectory_tracking.pdf`
Five stacked joint panels; each overlays measured + FNN + Physics-Reg. + EDR
over one selected test trajectory (savgol). Trajectory chosen by
`trajectory_select` (None=auto / int / geometry name). *Data:* champion
predictions. *Table:* `trajectory_tracking.csv`.

## Tables (`tables/*.csv`, raw data)

- **`champion_selection.csv`** — per arch × basis (global / full_data_train /
  full_data_val / full_data_test): run_id, data_fraction, params, train/val/test
  RMSE, best_val_epoch. (Simple scalars ⇒ CSV only, no chart.)
- **`headline_metrics.csv`** — one row/champion: traj-macro & pooled RMSE,
  R²_overall, R²_mean, per-joint RMSE J1–J5, params, epochs_ran,
  worst_traj_rmse. (Training wall-time was not recorded for these runs, so there
  is no training-time figure; `epochs_ran` is the available cost proxy.)
- **`grid_summary.csv`** — per arch: n_runs, best/mean/std/median test RMSE.
- **`grid_runs.csv`** — every grid run: n, architecture, status, test_rmse,
  elapsed_sec, data_train_fraction, seed.
- **`data_efficiency.csv`** — architecture, data_fraction_pct, test_rmse,
  test_rmse_std, n_seeds, val_rmse, gen_gap.
- **`capability_profile.csv`** — architecture, axis, raw_value, norm_score,
  lower_is_better (the 5 radar axes × 3 champions; raw + normalised).
- **`training_curves.csv`** — long form: architecture, epoch, train_rmse,
  val_rmse, overfit_gap (the 3 champions).
- **`edr_correction_evolution.csv`** — epoch + the 4 EDR δ-term magnitudes.
- **`per_joint_metrics.csv`** — architecture, joint, rmse, r2, mae, nrmse (3×5).
- **`trajectory_tracking.csv`** — geometry, sample, joint, measured, pred_fnn,
  pred_physreg, pred_edr for the selected trajectory.
